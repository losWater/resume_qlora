# 使用说明 —— 这个程序怎么用

本程序做两件事：
1. **简历结构化抽取**：输入一段中文简历纯文本 → 输出规范 JSON（姓名/学历/技能/工作年限/经历）。
2. **岗位匹配**：再给一段岗位 JD → 输出 0~100 匹配分 + 逐项理由。

> 前提：环境已搭好、模型已下载、已训练出 adapter（见《开发流程.md》）。
> 所有命令用项目自带的虚拟环境 python：`D:\cv_view\resume_qlora\.venv\Scripts\python.exe`

---

## 一、最常用：CLI Demo

### 1. 交互模式（推荐，演示用）

```powershell
cd D:\cv_view\resume_qlora
.\.venv\Scripts\python.exe cli_demo.py
```

流程：
1. 程序先加载模型（基座 + LoRA adapter，约十几秒）。
2. 提示「请粘贴简历文本」→ 粘贴简历 → **输入一个空行**结束。
3. 屏幕打印结构化简历 JSON。
4. 提示「（可选）粘贴岗位 JD」→ 粘贴 JD（或直接空行跳过）。
5. 若粘了 JD，打印匹配分 + 理由。
6. 问「再来一条？(y/N)」→ 输入 `y` 继续，其它退出。

### 2. 文件模式（批量/脚本调用）

把简历、JD 各存成一个 txt，然后：

```powershell
.\.venv\Scripts\python.exe cli_demo.py --resume 简历.txt --jd 岗位.txt
```

只想抽取、不做匹配，就只传 `--resume`：

```powershell
.\.venv\Scripts\python.exe cli_demo.py --resume 简历.txt
```

## 三、匹配分怎么算（规则，可解释）

总分 = 技能分 × 60% + 年限分 × 20% + 学历分 × 20%，满分 100。

- **技能分** = JD 要求技能里被简历命中的比例（模糊匹配：互相包含即算命中）。
- **年限分** = 简历年限 ≥ JD 要求 → 满分；否则按比例。
- **学历分** = 简历最高学历 ≥ JD 要求 → 满分；否则 0。

权重可在 `matcher.py` 顶部的 `WEIGHTS` 改。

## 五、常见问题

- **中文乱码**：命令行先执行 `chcp 65001`，或脚本里已加 UTF-8 强制输出，一般不会乱。
- **加载慢/显存紧**：关掉占显存的程序（浏览器、QQ 等）。模型 4-bit 加载约需 1.5~2GB 显存。
- **输出 JSON 偶尔字段抽错**：1.5B 小模型的正常局限（见 README「已知局限」）。
